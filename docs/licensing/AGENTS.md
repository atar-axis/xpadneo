# DOX Framework

## Purpose

This subtree owns license tracking and reuse documentation.


## Ownership

- Licensing notes and compliance references.
- Reuse metadata documentation.
- Explanations of project license boundaries.


## Local Contracts

- Keep license statements aligned with `LICENSE.md`, `LICENSES/`, and existing
  SPDX markers.
- Do not change license classifications as part of unrelated code or
  documentation edits.
- Treat license metadata changes as policy-sensitive and make them explicit in
  commit messages.


## Work Guidance

- Prefer precise SPDX identifiers over prose license descriptions.
- Check existing file headers before documenting a license rule.


## Verification

- Cross-check license references against the corresponding files and license
  texts.


## Child DOX Index

No child guides are currently needed.
